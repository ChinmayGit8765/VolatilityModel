## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → api (HTTP, loopback) | Untrusted path param `symbol` crosses here |
| api container → MLflow server (HTTP) | Champion model (pickle) loaded via alias proxy |
| api container → data bind mount | Reads processed/feature parquets; writes prediction log |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Tampering | path-traversal via `/forecast/{symbol}` | mitigate | Validate `symbol in KNOWN_ASSETS` before any filesystem access; reject unknown with 404 (V5) |
| T-03-10 | Information Disclosure | error responses | mitigate | Generic `detail` in 4xx/5xx; never expose stack traces or filesystem paths (V7) |
| T-03-11 | Tampering | pyfunc.load_model pickle deserialization | mitigate | Load only from this project's own registry by `@champion` alias; provenance via Plan 02 lineage tags |
| T-03-12 | Information Disclosure | api network exposure | mitigate | `ports: 127.0.0.1:8000:8000` only — never `0.0.0.0`; matches existing stack loopback posture; no auth needed for local portfolio scope (documented, V2/V4 N/A) |
| T-03-13 | Information Disclosure | prediction log file | accept | data/predictions/ bind-mounted inside compose, not exposed via HTTP; acceptable for Phase 3 local scope |
</threat_model>
