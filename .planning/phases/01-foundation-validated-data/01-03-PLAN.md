<objective>
Build the full validation layer: Pandera schemas for crypto and equity OHLCV, calendar-aware gap detection (crypto 24/7 vs equity XNYS sessions), stale-row and OHLC-consistency checks, and a single `validate_asset` dispatch entry point that quarantines offending rows and fails closed. After this plan, any data with gaps, bad ticks, stale rows, schema violations, or fabricated weekend equity bars is rejected before it can reach the feature pipeline.

Purpose: The Pandera schemas are the executable specification of what "clean data" means — they are the trust boundary the entire downstream stack relies on. Per RESEARCH, schemas are written as the ingest spec, not as a cleanup pass.

Output: crypto + equity schemas, calendar-aware checks, a `validate_asset` dispatcher, fixtures of intentionally-broken data, and unit tests proving each gate rejects the right failures.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| raw ingested OHLCV -> validation gate | The primary trust boundary: untrusted raw data is rejected here before any downstream phase consumes it |
| validation gate -> processed store / features | Only schema-and-calendar-clean data crosses; everything else is quarantined |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | gaps / missing sessions in OHLCV | mitigate | `crypto_gap_check` (24/7 date_range) and `equity_session_check` (XNYS sessions_in_range) reject missing days/sessions |
| T-03-02 | Tampering | fabricated weekend/holiday equity rows | mitigate | `equity_session_check` rejects any index not in XNYS sessions (RESEARCH Pitfall 2) |
| T-03-03 | Tampering | bad ticks (high<low, non-positive prices) | mitigate | Pandera positivity + OHLC-consistency checks reject before persist |
| T-03-04 | Tampering | stale / repeated-close feed | mitigate | `stale_row_check` flags excessive duplicate closes |
| T-03-05 | Information Disclosure | quarantine CSV contents | accept | Quarantine holds only OHLCV values already in the local store; written under data/quarantine/ (gitignored), no secrets |
| T-03-06 | Repudiation | silent validation bypass | mitigate | Gate fails closed (raises) on any failure; no path persists invalid data — proven by tests |
</threat_model>
