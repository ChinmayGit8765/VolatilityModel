<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo -> CI runner | Untrusted PR code runs in CI; must never reach live external APIs or secrets |
| host filesystem -> Docker containers | Windows NTFS/WSL2 boundary; CRLF and SQLite-lock failure modes live here |
| local secrets (.env) -> git | Postgres credentials must never be committed |
| validation gate -> data/processed | Only validated data is promoted; the gate must fail closed in the wired pipeline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | Postgres credentials in compose/.env | mitigate | Only `.env.example` with placeholders is committed; `.env` gitignored; env substitution `${VAR:-default}` keeps real values out of git |
| T-04-02 | Tampering | SQLite-on-Windows-bind-mount corruption | mitigate | Postgres backend with `postgres_data` named volume (never a bind mount), per RESEARCH Pitfall 4 |
| T-04-03 | Tampering | CRLF line endings break container entrypoints | mitigate | `.gitattributes` `*.sh text eol=lf` + container-bound file rules (RESEARCH .gitattributes example) |
| T-04-04 | Information Disclosure | CI making live API calls / leaking | mitigate | `VOLFORECAST_NO_LIVE_API=1` in CI; tests are fixture-only; adapters raise if the guard is set |
| T-04-05 | Tampering | dependency drift between local and CI | mitigate | `uv.lock` committed; CI uses `uv sync --locked` (RESEARCH Pitfall 5) |
| T-04-06 | Tampering | psycopg2-binary [ASSUMED] package | mitigate | Blocking human-verify legitimacy checkpoint (Task 2) before the compose backend relies on it |
| T-04-07 | Elevation of Privilege | exposed MLflow/Prefect ports | accept | Local-only single-user stack bound to localhost; no public exposure in v1 (cloud deploy is out of scope) |
</threat_model>
