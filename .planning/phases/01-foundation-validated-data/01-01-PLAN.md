<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public exchange API -> ingest process | Untrusted OHLCV data crosses here (bad ticks, gaps, forming candles) |
| ingest process -> parquet store | Data persisted to local filesystem; consumed by all downstream phases |
| local repo -> git | `.dvc` pointers and config committed; data and secrets must never be committed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | ingested OHLCV from public exchange | mitigate | Pandera `crypto_ohlcv_schema` gate runs before any persist; OHLC-consistency + positivity checks reject bad ticks; failures quarantined, not stored |
| T-01-02 | Tampering | forming/partial last candle | mitigate | `drop_incomplete_candles` excludes candles where open_time + timeframe_ms > now_ms; test enforces last stored date < today UTC |
| T-01-03 | Information Disclosure | exchange credentials | accept | ccxt public OHLCV needs no API key; no secrets handled in this plan; no `.env` created or committed |
| T-01-04 | Tampering | malicious parquet (pickle gadgets) | mitigate | Read only from own store / committed fixtures; Pandera validates schema before any computation |
| T-01-SC | Tampering | uv package installs | mitigate | All packages [VERIFIED] in RESEARCH Package Legitimacy Audit except psycopg2-binary and tenacity [ASSUMED]; `uv.lock` hash-locks transitive deps (see note) |

> Package legitimacy note: psycopg2-binary and tenacity are tagged [ASSUMED] (slopcheck unavailable). Both are decade-old, multi-million-weekly-download libraries with known source repos (github.com/psycopg/psycopg2, github.com/jd/tenacity). Neither is exercised by this plan's test code paths — psycopg2 is for the compose Postgres backend (Plan 04) and tenacity for yfinance retries (Plan 02). The blocking human-verify legitimacy checkpoints for these installs live in Plan 02 and Plan 04 where they are first exercised. This plan only resolves them into `uv.lock` as declared dependencies.
</threat_model>
