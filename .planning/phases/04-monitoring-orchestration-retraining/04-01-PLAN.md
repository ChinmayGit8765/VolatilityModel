<objective>
Build the feedback-loop labeller (MON-01): join arrived realized volatility against the logged forecasts to produce an append-only `data/monitoring/forecast_vs_realized.parquet`, AND log GARCH(1,1) live forecasts into the same table so the Wave-2 performance monitor can compare champion vs GARCH rolling QLIKE.

Purpose: This is the first vertical slice of the feedback loop — after this plan, a real operator can run one command and see forecast-vs-realized rows accumulate, with both the champion and the GARCH baseline scored on identical (asset, as_of_date) rows. This is the single most load-bearing extension in Phase 4: without GARCH live rows, the performance-degradation monitor (Plan 03) has nothing to compare against.

Output: `src/volforecast/monitoring/labeller.py`, `scripts/run_labeller.py`, `tests/unit/test_labeller.py`, and the runtime artifact `data/monitoring/forecast_vs_realized.parquet`.
</objective>
